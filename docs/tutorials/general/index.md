# General Tutorials

Skills that every team member benefits from, regardless of discipline — the
collaborative and organizational glue that makes game jams and semester projects
run smoothly.

---

## Topics Covered Here

This section is home to tutorials on subjects like:

- **Git basics** — cloning, committing, pushing, pulling, resolving merge conflicts
- **Git for game jams** — branching strategy, handling large binary files with Git LFS
- **GitHub workflow** — pull requests, code review, issues, project boards
- **Setting up your environment** — Unity Hub, VS Code + extensions, Godot, Git, GitHub Desktop
- **Project scoping** — estimating what's actually achievable in a jam or a semester
- **Team roles** — how to split work across programmers, artists, designers, and audio
- **Communication** — how to give and receive design feedback, async vs. sync collaboration
- **Postmortems** — why to do them, how to run them, what makes a good one
- **Playtesting sessions** — running a playtest, note-taking, synthesizing feedback
- **Game jam strategy** — Ludum Dare, Global Game Jam, how to prepare and what to expect
- **Open source contributing** — forks, PRs, reviewing other people's docs changes
